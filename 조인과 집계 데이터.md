__조인이란?__
<pre>
<code>
-- 2개 이상의 테이블에 있는 정보 중 사용자가 필요한 집합에 맞게 가상의 테이블처럼 만들어서 결과를 보여주는 일
</code>
</pre>
__조인의 종류__
<pre>
<code>
INNER JOIN : 특정 컬럼을 기준으로 정확히 매칭된 집합을 출력한다.
OUTER JOIN : 특정 컬럼을 기준으로 매칭된 집합을 출력하지만 한쪽의 집합은 모두 출력하고 다른 한쪽의 집합은 매칭되는 컬럼의 값 만 출력한다.
SELF JOIN : 동일한 테이블끼리의 특정 컬럼을 기준으로 매칭되는 집합을 출력한다.
FULL OUTER JOIN : INNER, LEFT, OUTER, RIGHT OUTER 조인 집합을 모두 출력한다.
CROSS JOIN : CARTESIAN PRODUCT이라고도 하며 조인되는 두 테이블에서 곱집합을 반환한다.
NATURAL JOIN : 특정 테이블의 같은 이름을 가진 컬럼 간의 조인집합을 출력한다.
</code>
</pre>

__INNER 조인__
<pre>
<code>
-- 특정 컬럼을 기준으로 정확히 매칭된 집합을 출력한다. INNER 조인은 대표적인 조인의 종류
</code>
</pre>
<pre>
<code>
SELECT
	A.CUSTOMER_ID,
	A.FIRST_NAME,
	A.LAST_NAME,
	A.EMAIL,
	B.AMOUNT,
	B.PAYMENT_DATE
FROM
	CUSTOMER A					--CUSTOMER 테이블과 PAYMENT 테이블을 CUSTOMER_ID 기준으로 조인한다.
INNER JOIN PAYMENT B ON
	A.CUSTOMER_ID = B.CUSTOMER_ID ;
WHERE A.CUSTOEMR_ID = 2			--CUSTOMER_ID가 2인 집합만을 출력한다.
</code>
</pre>
<pre>
<code>
--고객 여러건의 결제를 할수있다. 고객 1 : 결제 M 1:M 관계
</code>

<code>

	A.CUSTOMER_ID,
	A.FIRST_NAME,

	A.EMAIL,

	C.FIRST_NAME AS S_FIRST_NAME,
	C.LAST_NAME AS S_LAST_NAME

__LEFT OUTER 조인__
<pre>
<code>
SELECT
	A.ID AS ID_A,
	A.FRUIT AS FRUIT_A,
	B.ID AS ID_B,
	B.FRUIT AS FRUIT_B
FROM
	BASKET_A A					--BASKET_A 테이블과 BASKET_B 테이블을 조인하는데 BASKET_A를 기준집합으로 조인한다.
LEFT JOIN BASKET_B B ON
	A.FRUIT = B.FRUIT
WHERE B.ID IS NULL				--B.ID가 NULL 인 것만 출력한다.
</code>
</pre>

__HAVING 절__
<pre>
<code>
--GROUP BY 절과 함께 HAVING 절을 사용하여 GROUP BY의 결과를 특정 조건으로 필터링 하는 기능을 한다.
    </code>
</pre>
<pre>
<code>
SELECT 
	COLUMN_1,				--GROUP BY 컬럼 기재
	집계함수(COLUMN_2)			--집계함수 사용
FROM 
	TABLE_NAME
GROUP BY COLUMN_1			--GROUP BY 절 기재, N개의 컬럼을 GROUP BY하는 경우 ','구분
HAVING 조건식;				--GROUP BY 절을 FROM 또는 WHERE절 바로 뒤에 나타나야함 

    </code>
</pre>
<pre>
<code>
--HAVING 절은 GROUP BY 절에 의해 생성된 그룹행의 조건을 설정한다.
--반면에 WHERE 절은 GROUP BY 절이 적용되기 전에 개별 해의 조건을 설정한다.
    </code>
</pre>
---------------------------------------------------------------------------------------------------------
<pre>
<code>
SELECT 
	CUSTOMER_ID,
	SUM(AMOUNT) AS AMOUNT_SUM
FROM 
	PAYMENT
GROUP BY CUSTOMER_ID
HAVING SUM(AMOUNT) > 200;
    </code>
</pre>
---------------------------------------------------------------------------------------------------------

<code>

	PAYMENT A,
	CUSTOMER B
WHERE
	A.CUSTOMER_ID = B.CUSTOMER_ID

	A.CUSTOMER_ID,
	B.EMAIL

	SUM(A.AMOUNT) > 200
     </code>

	SEGMENT,

FROM
	SALES				--SALES 테이블을 조회한다.
GROUP BY
	GROUPING SETS ( 
	(BRAND,SEGMENT),	--BRAND, SEGMENT 컬럼 기준으로 합계를 구한다.
	(BRAND),		--BRAND 컬럼 기준으로 합계를 구한다.
	(SEGMENT),		--SEGMENT 컬럼 기준으로 합계를 구한다.
	()				--테이블 전체를 기준으로 합계를 구한다.
); 
   </code>

<pre>
<code>
SELECT 
	GROUPING(BRAND) GROUPING_BRAND,					--해당 컬럼이 집계에 사용되었으면 0, 그렇지 않으면 1을 리턴한다.
	GROUPING(SEGMENT) GROUPING_SEGMENT,
	BRAND,
	SEGMENT,
	SUM(QUANTITY)
FROM 
	SALES
GROUP BY
GROUPING SETS 
(
	(BRAND,SEGMENT),
	(BRAND),
	(SEGMENT),
	()
)
ORDER BY BRAND, SEGMENT;
   </code>

<code>
--지정된 GROUPING 컬럼의 소계를 생성하는데 사용된다. 간단한 문법으로 다양한 소계를 출력할 수 있따.
   </code>
</pre>
<pre>
<code>
SELECT
	C1,
	C2,
	C3,
	집계함수(C4)
FROM
	TABLE_NAME
GROUP BY
	ROLLUP (C1,C2,C3);
     </code>
</pre>
<pre>
<code>
--소계를 생성할 컬럼을 지정한다. 컬럼 지정 순서에 따라 결과값이 달라질 수 있다.(제일 앞에 놓인 것에 소계를 구함)
   </code>
</pre>

--------------------------------------------------------------------------------------------------------------
<pre>
<code>
SELECT 
	C1,C2,C3,집계함수(C4)
FROM TABLE_NAME
GROUP BY C1
ROLLUP (C2,C3);		--특정 컬럼은 제외한 부분적인 ROLLUP도 가능하다.
   </code>
</pre>

--------------------------------------------------------------------------------------------------------------
<pre>
<code>
SELECT 
	BRAND,
	SEGMENT,
	SUM(QUANTITY)
FROM 
	SALES
GROUP BY
	ROLLUP(BRAND,SEGMENT)		--BRAND, SEGMENT 컬럼 기준으로 ROLL UP 한다.
ORDER BY 
	BRAND, SEGMENT;
     </code>
</pre>

<pre>
<code>
--GROUP BY 별 합계 + ROLLUP 절에 맨 앞에 쓴 컬럼 기준의 합계 + 전체 합계
     </code>
</pre>
--------------------------------------------------------------------------------------------------------------
<pre>
<code>
SELECT 
	SEGMENT,
	BRAND,
	SUM(QUANTITY)
FROM 
	SALES
GROUP BY SEGMENT,
ROLLUP (BRAND)
ORDER BY 
	SEGMENT, BRAND
      </code>
</pre>
<pre>
<code>
--부분 ROLLUP 시 전체 합계는 구하지 않는다. GROUP BY 별 합계 + 맨앞에 쓴 컬럼별 합계
     </code>
</pre>
---------------------------------------------------------------------------------------------------------------
__CUBE 절 문법__
<pre>
<code>
--지정된 GROUPING 컬럼의 다차원 소계를 생성하는데 사용, 간단한 문법으로 다차원 소계 출력
     </code>
</pre>
<pre>
<code>
SELECT
	C1,
	C2,
	C3,
	집계함수(C4)
FROM
	TABLE_NAME
GROUP BY
	CUBE(C1,C2,C3);		--CUBE절에서 다차원 소계를 할 열을 지정한다. (지정한 그룹의 모든 경우에 수에 대한 소계와 총계를 구한다.)
       </code>
</pre>

----------------------------------------------------------------------------------------------------------------
<pre>
<code>
SELECT
	C1,
	C2,
	C3,
	집계함수(C4)
FROM
	TABLE_NAME
GROUP BY
	C1,
	CUBE(C2,C3);
       </code>
</pre>
<pre>
<code>
--특정 컬럼만 분리하여 CUBE 지정을 할 수 있다.
      </code>
</pre>
----------------------------------------------------------------------------------------------------------------
<pre>
<code>
SELECT
	BRAND,
	SEGMENT,
	SUM(QUANTITY)
FROM
	SALES
GROUP BY
	CUBE(BRAND,SEGMENT)
ORDER BY
	BRAND, SEGMENT;	
        </code>
</pre>
<pre>
<code>
--CUBE = GROUP BY 절 합계 + BRAND 별 합계 + SEGMENT 별 합계 + 전체합계
        </code>
</pre>
--------------------------------------------------------------------------------------------------------------------
<pre>
<code>
SELECT
	BRAND,
	SEGMENT,
	SUM(QUANTITY)
FROM
	SALES
GROUP BY
	BRAND,
	CUBE(SEGMENT)
ORDER BY
	BRAND,
	SEGMENT;	
  
          </code>
</pre>
<pre>
<code>
--부분 cube = group by별 합계 + 맨앞에쓴 컬럼별 합계
          </code>
</pre>
